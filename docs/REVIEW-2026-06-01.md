## Findings by priority

### P0 — Block release

> None. Build, tests, signing, and core flows all green.

### P1 — Fix before publish

#### P1-1 — AdMob IDs are still Google's TEST unit IDs

- **Where:** `lib/src/core/ads/ad_helper.dart:46-51`, `android/app/build.gradle.kts:42-44`, `ios/Runner/Info.plist:28`
- **What:** All five ad unit IDs and the app ID are `ca-app-pub-3940256099942544/...` (Google's official test IDs).
- **Impact:** Live builds will serve test ads forever, and the AdMob dashboard will show zero real revenue data.
- **Fix:** B4 in the release checklist — 15 min swap of the IDs for real ones once AdMob account exists.
- **Severity:** P1 because if you publish without swapping, you'll get reports of "ads say Test Ad" from users.

#### P1-2 — Keystore password is TEMP and stored in plaintext in two places

- **Where:** `android/key.properties` and `/tmp/niduna_temp_keystore_pwd.txt`
- **What:** The keystore was generated with a temporary password for this dev cycle. The password is checked into `android/key.properties` (which is gitignored, OK) and also written to `/tmp/niduna_temp_keystore_pwd.txt` (not gitignored, but `/tmp` isn't tracked).
- **Impact:** If `/tmp/niduna_temp_keystore_pwd.txt` is leaked, the keystore is compromised. Same for `android/key.properties` if it ever gets included by mistake.
- **Fix:** Rotate via `keytool -storepasswd` + `keytool -keypasswd`, then delete the temp file. Document the rotation steps in `docs/RELEASE_COMMANDS.md` (already there). 5 min.
- **Severity:** P1 because the cert is used for 27 years and a compromise now is permanent.

#### P1-3 — Privacy policy URL not hosted (B5 blocked)

- **Where:** Settings screen, B5 in release checklist.
- **What:** Code is ready to add a privacy policy row but the URL is missing.
- **Impact:** Play Store requires a privacy URL in Data Safety / Store Listing. Without it, submission is rejected.
- **Fix:** Host a privacy page (GitHub Pages, Cloudflare Pages, or just a gist). C1 in the release checklist. ~1 hr.
- **Severity:** P1 because it's a hard block on submission.

### P2 — Quality / maintainability

#### P2-1 — Test coverage is uneven

- 60.3% aggregate line coverage is OK for a UI app, but most "controller" and "view" files are 0% tested (they're tested via integration tests on the iOS sim, but `flutter test --coverage` doesn't run integration tests).
- **Where to improve:** widget tests for `app_shell.dart`, `convert_screen.dart`, `settings_screen.dart`, `charts_screen.dart`. Pure widget tests with mock controllers would be cheap to add and would raise coverage meaningfully.
- **Effort:** ~3-4 hours of adding widget tests.
- **Severity:** P2 — nice-to-have, not blocking.

#### P2-2 — 17 silent `catch (_)` blocks across the rates/ad layers

- **Where:** `lib/src/core/rates/crypto/*`, `lib/src/core/rates/cache/*`, `lib/src/core/ads/*`
- **What:** These swallow exceptions silently when cache writes fail or ads fail to load. The pattern is intentional (don't crash on ad/cache failure) but means a misbehaving network or corrupt cache has no diagnostic trail.
- **Recommendation:** Replace the silent catches with at least one `debugPrint` or a `Sentry.report(...)` (when you add observability). Or document in a code comment that the silent catch is intentional.
- **Effort:** ~30 min.
- **Severity:** P2 — observability, not blocking.

#### P2-3 — `RELEASE_CHECKLIST.md` is stale

- **Where:** Top-level `RELEASE_CHECKLIST.md` "Blocker Summary"
- **What:** B1 (Generate release keystore), B2 (Create key.properties), B3 (Update build.gradle.kts), B6 (Build release AAB) are all marked ❌ but are actually done in commits `200c888` and the work we did this session.
- **Recommendation:** Update the file to reflect actual state. The "Done ✅" section at the bottom is already there but the top "Blocker Summary" is stale.
- **Effort:** ~10 min.
- **Severity:** P2 — documentation accuracy.

#### P2-4 — CoinPaprika is referenced as a dev-only provider but the docs say "publish-time" is sensitive

- **Where:** `docs/providers/coinpaprika.md` and `lib/src/core/rates/provider_factory.dart`
- **What:** CoinPaprika is correctly tagged `dev-only`. The build system defaults to `dev_coinpaprika` profile for local dev. The release `release_safe` profile uses Frankfurter + fawazahmed0.
- **Status:** Correct as-is, but worth a quick re-read to confirm no other dev-only provider sneaks into the release build. (No issue found on re-read — `provider_factory.dart` correctly switches based on `PROVIDER_PROFILE`.)
- **Severity:** P2 — verified, no action needed.

- **Re-enable:** `GEM_HOME=/opt/homebrew/Cellar/cocoapods/1.16.2_2/libexec ruby ios/scripts/add_widget_target.rb` re-runs the widget setup script (idempotent). I ran this before writing the report, so main is consistent with what was merged.
- **Severity:** P3 — informational.

## Architecture review

### State management: Provider + ChangeNotifier

- 6 controllers, all `ChangeNotifier`. 23 files use `Provider` / `context.watch` / `context.read`.
- **Consistent.** No mixing of Riverpod / Bloc / Redux.
- `app_shell.dart` is 204 lines (the largest after generated localizations). It wires 7 controllers and the app's root structure. Could be split into a `MultiProvider` constructor, but not urgent.
- **Pattern verdict:** Standard, well-understood, easy to onboard.

### Error handling

- 26 try/catch blocks. 17 are `catch (_)` (silent). 9 are named.
- **Pattern verdict:** Mostly intentional (ads/cache shouldn't crash the app). Worth adding a `debugPrint` or observability call instead of `_` in 5-10 high-value places (cache write failures, ad load failures).

### Test organization

- `test/` mirrors `lib/src/features/<feature>/` reasonably well.
- `test/widget_test.dart` is a kitchen-sink file (180+ tests across many widgets). Could be split per-feature for faster test runs and easier maintenance.
- **Pattern verdict:** Functional but a bit monolithic. Not urgent.

### Code smells observed

- 7 files ≥ 200 lines (mostly widgets and theme files, not a concern for Dart).
- 0 `TODO` / `FIXME` / `XXX` / `HACK` markers in `lib/` — clean.
- 1 unused import in `lib/src/core/widget/home_widget_provider.dart` (fixed in this review).
- 2 `unnecessary_underscores` in `visible_rates_list.dart` (fixed in this review).
- 1 redundant `foundation.dart` import in `lib/main.dart` (fixed in this review).
- **Pattern verdict:** Clean. Code is well-maintained.
